# ToDoリストを管理するAPI
## 概要
- ToDoリストのCRUD（作成、読み取り、更新、削除） をカバーするAPI

## 詳細
- 下記4種類のAPIを実装する

### 1. 作成
- メソッド `POST`
- エンドポイント `/todos`
- 詳細
  - ToDo情報（タイトルと優先度）を受け取り、新しいデータを追加する。
  - 【成功時】ステータスコード201（Created）を返し、追加されたToDoのID・タイトル・優先度・ステータス（固定値：未着手）をJSON形式で返す。
  - 【失敗時】ステータスコード500（サーバーエラー）を返す。

### 2. 読み取り
- メソッド `GET`
- エンドポイント `/todos`
- 詳細
  - 全ToDoのデータを取得する。
  - 【成功時】ステータスコード200（OK）を返し、すべてのToDoをJSON形式の配列で返す。
  - 【失敗時】ステータスコード500（サーバーエラー）を返す。

### 3. 更新
- メソッド `PUT`
- エンドポイント `/todos/:id`
- 詳細
  - ToDo情報（タイトル、優先度、ステータス）とIDを受け取り、指定したIDのToDo情報を更新する。
  - 【成功時】ステータスコード200（OK）を返し、更新後のToDo情報（ID・タイトル・優先度・ステータス）をJSON形式で返す。
  - 【対象ToDoが存在しない場合】ステータスコード404（Not Found）を返す。
  - 【失敗時】ステータスコード500（サーバーエラー）を返す。

### 4. 削除
- メソッド `DELETE`
- エンドポイント `/todos:id`
- 詳細
  - 指定したIDのToDoを削除する。
  - 【成功時】ステータスコード200（OK）を返し、削除完了メッセージをJSON形式で返す。
  - 【対象ToDoが存在しない場合】ステータスコード404（Not Found）を返す。
  - 【失敗時】ステータスコード500（サーバーエラー）を返す。
