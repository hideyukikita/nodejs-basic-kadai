# ToDo API テスト仕様書

## 1. 作成 (POST /todos)
**正常系**
- 内容: 新規ToDoの登録
- ボディ: `{"title": "買い物", "priority": 1}`
- 期待: `201 Created`
- レスポンス: `{"id": 1, "title": "買い物", "priority": 1, "status": "未着手"}`

**異常系**
- 内容: DB接続エラー等
- 期待: `500 Internal Server Error`

---

## 2. 読み取り (GET /todos)
**正常系**
- 内容: 全データの取得
- 期待: `200 OK`
- レスポンス: `[{"id": 1, "title": "...", ...}]`（配列）

**異常系**
- 内容: 取得失敗
- 期待: `500 Internal Server Error`

---

## 3. 更新 (PUT /todos/:id)
**正常系**
- 内容: ID指定でデータを更新
- ボディ: `{"title": "掃除", "priority": 2, "status": "完了"}`
- 期待: `200 OK`
- レスポンス: 更新後のデータオブジェクト

**異常系 (404)**
- 内容: 存在しないIDを指定
- 期待: `404 Not Found`
- レスポンス: `{"error": "更新対象のデータが見つかりません。"}`

**異常系 (500)**
- 内容: 更新処理失敗
- 期待: `500 Internal Server Error`

---

## 4. 削除 (DELETE /todos/:id)
**正常系**
- 内容: ID指定でデータを削除
- 期待: `200 OK`
- レスポンス: `{"message": "データの削除が成功しました。"}`

**異常系 (404)**
- 内容: 存在しないIDを指定
- 期待: `404 Not Found`

---

## 5. 実行用コマンド (curl)

### [POST] 作成
```bash
curl -X POST -H "Content-Type: application/json" \
-d '{"title":"テスト","priority":1}' \
http://localhost:3000/todos
{"error": "データの削除に失敗しました。"}
